    Tema 1

    Base de datos, SGBD y arquitectura

    Base de datos

    Es un conjunto organizado de datos relacionados entre sí, persistentes y accesibles por un sistema.

    SGBD

    El Sistema Gestor de Base de Datos se encarga de almacenar, gestionar y proteger los datos.

    Arquitectura

    Normalmente se estructura con capas de interfaz, motor, almacenamiento y optimización de consultas.

    DDL · DML · DCL · TCL

    DDL define estructuras, DML manipula datos, DCL controla permisos y TCL gestiona transacciones.

    Entidades y atributos

    La entidad representa un concepto del mundo real; los atributos describen sus propiedades.

    Relaciones y cardinalidad

    Las relaciones enlazan entidades y la cardinalidad define cuántas ocurrencias participan.

    Entidades débiles

    Dependen de otra entidad para existir y necesitan una clave parcial o discriminante.

    Generalización y especialización

    Se usan para modelar jerarquías y clasificar entidades con relaciones de herencia.

    Tema 3

    Modelo Relacional y Álgebra Relacional

    Conceptos

    Relación: tabla; tupla: fila; atributo: columna; dominio: conjunto de valores válidos.

    Claves

    La clave primaria identifica un registro, la clave foránea referencia otra tabla y las canditadas son alternativas.

    ER → Relacional

    Una entidad se convierte en tabla; un atributo pasa a columna; una relación puede producir clave foránea.

    Álgebra relacional

    Selección, proyección, unión, intersección, diferencia, producto cartesiano, join y división.

    Tema 4

    DDL con constraints

    SQL 
    CREATE DATABASE gestion_ventas;
GO

CREATE TABLE clientes (
    id_cliente INT PRIMARY KEY IDENTITY(1,1),
    nombre NVARCHAR(100) NOT NULL,
    email NVARCHAR(120) UNIQUE,
    edad INT CHECK (edad >= 18),
    fecha_alta DATETIME DEFAULT GETDATE()
);

ALTER TABLE clientes ADD telefono NVARCHAR(15);
ALTER TABLE clientes ALTER COLUMN nombre NVARCHAR(120) NOT NULL;

DROP TABLE clientes;
TRUNCATE TABLE clientes;

    Comandos esenciales

    CREATE, ALTER, DROP y TRUNCATE permiten definir, modificar y borrar objetos de la base de datos.

    Constraints

    PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, DEFAULT y NOT NULL garantizan integridad y consistencia.

    Tema 5

    Consultas SQL

    SQL 
    SELECT
    c.nombre,
    COUNT(p.id_pedido) AS total_pedidos
FROM clientes c
LEFT JOIN pedidos p ON p.id_cliente = c.id_cliente
WHERE c.activo = 1
GROUP BY c.nombre
HAVING COUNT(p.id_pedido) >= 1
ORDER BY total_pedidos DESC;

    Cláusulas y funciones

    SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY, DISTINCT, TOP, LIKE, BETWEEN, IN, IS NULL, JOIN y funciones agregadas.

    Joins y subconsultas

    Los joins combinan filas entre tablas; las subconsultas encapsulan lógica de filtrado o agregación.

    Tema 6

    INSERT, UPDATE, DELETE y SELECT INTO

    SQL 
    INSERT INTO clientes (nombre, email, activo)
VALUES ('Ana García', 'ana@correo.com', 1);

UPDATE clientes
SET activo = 0
WHERE id_cliente = 10;

DELETE FROM clientes
WHERE id_cliente = 12;

SELECT *
INTO clientes_backup
FROM clientes
WHERE activo = 1;

    Manipulación de datos

    INSERT añade filas, UPDATE modifica, DELETE elimina y SELECT INTO crea una nueva tabla a partir de una consulta.

    Buena práctica

    Usa WHERE en UPDATE/DELETE y valida siempre el resultado antes de confirmar la operación.

SET @mensaje = 'Inicio de proceso';

WHILE @contador < 3
BEGIN

        PRINT 'Llegó a 2';

        PRINT CONCAT('Valor actual: ', @contador);
END;

BEGIN TRY
    SELECT 1 / 0;
END TRY
BEGIN CATCH

END CATCH;

    Tema 8

    Procedimientos y funciones

    T-SQL 
    CREATE PROCEDURE sp_ventas_por_cliente
    @id_cliente INT,
    @total DECIMAL(18,2) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    SELECT @total = SUM(total)
    FROM pedidos
    WHERE id_cliente = @id_cliente;
END;

    Procedimientos

    Un procedimiento encapsula lógica y puede recibir parámetros de entrada o salida.

    Funciones

    Las funciones devuelven valores reutilizables y pueden ser escalares o de tabla.

    Tema 9

    Triggers

    T-SQL 
    CREATE TRIGGER trg_audita_pedidos
ON pedidos
AFTER INSERT, UPDATE
AS
BEGIN
    SET NOCOUNT ON;

    INSERT INTO auditoria_pedidos (id_pedido, accion, fecha)
    SELECT i.id_pedido, 'INSERT/UPDATE', GETDATE()
    FROM inserted i;
END;

    Tipos de trigger

    AFTER se ejecuta tras la operación y INSTEAD OF sustituye la acción original.

    inserted y deleted

    Estas tablas virtuales permiten inspeccionar los valores nuevos o antiguos afectados por la operación.

    Tema 10

    Cursores

    T-SQL 
    DECLARE c_ventas CURSOR FOR
SELECT id_cliente, total
FROM pedidos;

OPEN c_ventas;
DECLARE @id INT, @total DECIMAL(18,2);

FETCH NEXT FROM c_ventas INTO @id, @total;
WHILE @@FETCH_STATUS = 0
BEGIN
    PRINT CONCAT('Cliente: ', @id, ' Total: ', @total);
    FETCH NEXT FROM c_ventas INTO @id, @total;
END;

CLOSE c_ventas;
DEALLOCATE c_ventas;

    Flujo del cursor

    DECLARE CURSOR, OPEN, FETCH, WHILE, @@FETCH_STATUS, CLOSE y DEALLOCATE.

    Uso

    Se usan para procesar registros uno a uno, aunque hoy se prefieren soluciones de conjunto.

    MongoDB

    Introducción, colecciones, documentos y CRUD

    MongoDB

    MongoDB guarda documentos JSON-like en colecciones. Su modelo es flexible y orientado a documentos.

    CRUD y agregación

    find, insertOne, insertMany, updateOne, updateMany, deleteOne, deleteMany, aggregate y operaciones como sort, limit y skip.

    Índices

    Los índices aceleran consultas y reducen el coste de lectura, aunque aumentan el espacio de almacenamiento.

    Operadores

    Se usan para actualizar campos, comparar valores y construir filtros avanzados.

    Documentos

    Un documento MongoDB es un registro flexible con clave-valor, sin esquema rígido. Esto facilita la evolución del modelo.

    Collection

    Una colección agrupa documentos y suele contener una familia común. No exige una estructura exacta en todos los registros.

    lookup

    Permite hacer joins entre colecciones a nivel de agregación y es muy útil para modelos de documentos relacionados.

    JavaScript / MongoDB 
    db.ventas.find({ "estado": "pagado" })
  .sort({ fecha: -1 })
  .limit(5);

db.ventas.updateMany(
  { total: { $gte: 100 } },
  { $set: { prioridad: "alta" } }
);

db.ventas.aggregate([
  { $match: { estado: "pagado" } },
  { $group: { _id: "$cliente", total: { $sum: "$total" } } }
]);
    Extras

    Trucos, errores frecuentes y chuleta final

    Trucos

    Usa índices en columnas de filtrado, evita SELECT * y prueba la ejecución con EXPLAIN o PLAN.

    Errores frecuentes

    Olvidar WHERE en UPDATE/DELETE, confundir JOIN con CROSS JOIN, no controlar NULL y no usar alias claros.

    Chuleta final

    Recuerda: primero modela, luego normaliza, después define tablas y solo al final consulta y mantiene datos.
